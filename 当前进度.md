# AI Knowledge System — 当前进度

日期：2026-05-30

## 已完成

### 1. 架构重写（1号提交）
- 删除 `VectorStoreIndex([])` 和 llama_index 依赖
- 实现三层记忆系统：短期（最近20轮）+ 长期（Qdrant）+ 全局记忆（自动生成）
- LLM 适配层：抽象基类 + Ollama / DeepSeek / OpenAI 三套实现，一行 config 切换
- OpenAI 兼容 API：`POST /v1/chat/completions`
- 项目结构：rag-api/models/ rag-api/core/ rag-api/services/ rag-api/api/

### 2. 连通性修复（2号提交）
- docker-compose 添加 `OLLAMA_BASE_URL=http://host.docker.internal:11434` 和 `extra_hosts`
- 移除 `--reload` 模式（与包加载冲突）
- embedding 从 sentence-transformers 改为 Ollama API（因 PyTorch 版本兼容问题）
- 默认 embedding 模型：`nomic-embed-text:latest`（274MB）
- 默认 LLM 模型：`qwen3.5:9b`（6.6GB）
- Qdrant 客户端 API 从 `search()` 改为 `query_points()`（兼容新版 qdrant-client）
- 超时从 120s 逐步提升至 900s（冷启动加载模型需要）

### 3. 分层记忆（未提交的改动）
- 全局记忆变为全局共享（`__global__`），跨 session 保留
- 对话记忆按 `session_id` 过滤，各自独立
- 记忆检索同时查当前 session + 全局记忆，去重合并
- `POST /warmup` 预热端点（预加载 embedding + LLM 模型）

## 待修复问题

### P0 — OpenWebUI 无法通过 rag-api 对话
- 所有从 OpenWebUI 发往 rag-api 的请求均返回 `500 Internal Server Error`
- 原因是冷启动超时：首次请求需加载 nomic-embed-text（~60s）+ qwen3.5（~390s），总耗时 ~8 分钟
- 当前通过容器内预热可工作，但需确保预热完成
- **OpenWebUI 配置检查**：必须指向 `http://rag-api:8000/v1`，不能直连 Ollama

### P0 — OpenWebUI 直连 Ollama
- 用户能收到回复说明 OpenWebUI 可能直接连了本地 Ollama（localhost:11434）
- 需要确认 OpenWebUI 的连接配置，删除直连 Ollama 的连接

### P1 — 容器重启后 host.docker.internal 短暂断连
- 重启 rag-api 后偶尔出现 "Network is unreachable" 错误（持续几秒）
- 猜测是 Docker Desktop for Windows 的网络堆栈初始化时序问题
- 实测过几秒后自动恢复

### P2 — 清理与优化
- 删除容器镜像中的 sentence-transformers 和 torch 以减小体积
- 将来升级到 Qwen3-Embedding-8B，但需先解决 PyTorch 兼容问题
- 添加 streaming 支持（OpenWebUI 期待但当前未实现）
- prompt 中部分中文文本的 encoding 显示异常（`??,?????????` 等乱码）
