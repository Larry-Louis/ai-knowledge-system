# AI Knowledge System — 当前进度

日期：2026-05-31

## 已完成

### 1. 架构重写
- 删除 `VectorStoreIndex([])` 和 llama_index 依赖
- 实现三层记忆系统：短期（最近20轮）+ 长期（Qdrant）+ 全局摘要
- LLM 适配层：抽象基类 + Ollama / DeepSeek / OpenAI 三套实现
- OpenAI 兼容 API：`POST /v1/chat/completions`

### 2. LLM 切换为 DeepSeek
- 默认 LLM 从 Ollama(qwen3.5:9b) 切换为 DeepSeek(deepseek-v4-flash/pro)
- API Key 通过 `.env` 文件配置，docker-compose 自动读取
- 支持 Open WebUI 实时切换模型（flash ↔ pro）

### 3. 思维链展示
- 透传 DeepSeek `reasoning_content` 到 API 响应
- Open WebUI 原生灰色可折叠思维链展示

### 4. 分层记忆系统
- 新增 `layer` 字段区分记忆层
- 预设三层：`general`（默认）、`story`（故事创作）、`docreader`（文档分析）
- `core` 层始终生效，不随切换改变
- API: `GET/POST /role` 切换当前层
- 模型名后缀 `:story` 或直接写 `story` 自动切换

### 5. 文档分析服务 (doc-reader)
- 独立服务 `doc-reader`（端口 19000），与 rag-api 完全隔离
- 上传 .txt / .pdf，自动按章节分块存入 Qdrant `documents` 集合
- 对话式查询：提问后自动搜索全文档相关内容
- Web 前端界面挂载/卸载文档

### 6. 文档挂载集成
- rag-api 对话时自动搜索已挂载文档的关联内容
- `POST /documents/active` 管理挂载列表
- 相似度阈值 0.65，防止无关内容混入

### 7. 基础设施
- Docker Ollama 运行 nomic-embed-text（274MB）
- 移除 host.docker.internal 依赖，全部走 Docker 内部网络
- Qdrant 新增 `documents` 集合 + timestamp 索引

### 8. 调试辅助
- `GET /v1/last-prompt` 查看最近一次完整 prompt
- 支持 `GET /role` 查看当前层和可用层
