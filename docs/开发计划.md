# AI Memory System — 总体目标与分阶段开发计划



日期：2026-05-31



---



# 一、总体目标



将当前系统从：



```

聊天记录 + 向量检索（RAG）

```



升级为：



```

长期记忆操作系统（Memory OS）

=

Semantic Memory（语义知识）

+ Episodic Memory（事件经历）

+ Insight Memory（用户洞察）

+ Knowledge Graph（关系网络）

```



---



## 目标拆解



### 1. 能力目标



* 记住用户长期兴趣、项目与技术栈

* 记住关键历史对话与决策过程

* 能跨时间关联不同主题

* 自动总结用户行为与知识结构



### 2. 性能目标



* Prompt Token 减少 50%+

* 向量检索量减少 60%+

* 长期运行不退化

* 响应速度提升



---



# 二、Phase 1：Memory Optimization（基础优化层）



## 核心目标



让系统具备：



```

判断什么值得记住

控制记忆使用成本

优化检索范围

```



---



## 主要模块



### 1. Importance Score（重要性评分）



* 为每条记忆计算 0~1 分数

* 用于排序与过滤

* 影响最终 Prompt 构建



---



### 2. Layer Router（层路由）



将请求分流到不同记忆层：



* general

* story

* docreader

* core



减少无效检索范围



---



### 3. Context Budget（上下文控制）



* 限制最大 token

* 按 score 过滤记忆

* 防止 Prompt 膨胀



---



## 阶段结果



* Token 显著下降

* 检索范围缩小

* 响应速度提升



---



# 三、Phase 2：Insight Memory（记忆压缩层）



## 核心目标



从“存储信息”升级为“提炼用户画像”



---



## 主要模块



### 1. Insight Memory



自动总结：



* 长期兴趣

* 当前项目

* 技术栈

* 行为模式



---



### 2. Insight Builder



流程：



```

原始记忆 → LLM总结 → Insight生成 → 存储

```



---



## 阶段结果



* 100条碎片 → 5条洞察

* 用户画像稳定

* Token进一步下降



---



# 四、Phase 3：Episode Memory（事件记忆层）



## 核心目标



记录“发生了什么”，而不是“说了什么”



---



## 主要模块



### 1. Episode Generation



自动生成：



* 会话标题

* 摘要

* 关键结论

* 待办事项



---



### 2. Episode Storage



按会话或时间切分存储



---



## 阶段结果



* 可回溯历史讨论

* 支持长期项目跟踪

* 提升上下文连续性



---



# 五、Phase 4：Memory Distillation（记忆蒸馏层）



## 核心目标



控制记忆增长，避免无限累积



---



## 主要模块



### 1. Memory Compression



将多条 raw memory 压缩为：



* summary memory

* insight memory



---



### 2. Memory Lifecycle



```

active → compressed → archived

```



---



## 阶段结果



* 记忆规模可控

* 长期运行稳定

* Token成本下降



---



# 六、Phase 5：Knowledge Graph（关系网络层）



## 核心目标



建立“记忆之间的关系”



---



## 主要模块



### 1. Entity Extraction



抽取：



* 人

* 项目

* 技术

* 概念

* 事件



---



### 2. Relationship Graph



构建关系：



* DEVELOPS

* USES

* INTERESTED_IN

* RELATED_TO



---



### 3. Graph Storage



推荐：



* Kuzu（轻量嵌入式图数据库）



---



## 阶段结果



* 支持跨主题推理

* 建立长期知识网络

* 提升复杂问题理解能力



---
