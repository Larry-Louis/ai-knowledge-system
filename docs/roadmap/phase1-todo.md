# Phase 1 未实现项清单（按当前代码状态）

日期：2026-07-01

---

## 一、当前已完成基线（用于界定未实现项）

- Rule Score 三维框架已接入：结构分 + 语义相关 + 信息密度分。
- 第二维已从极性逻辑替换为实体驱动的信息密度逻辑（detect_information_density）。
- 信息密度已支持轻量 NLP 召回（spaCy 原型）并输出可解释 evidence。

> 以下仅列“仍未实现”与“需补齐”事项。

---

## 二、信息密度（第二维）剩余缺口

### 2.1 实体类型覆盖未补齐（P0）

当前可较稳定覆盖：
- Programming Language
- Framework
- API
- Tool
- Software
- Version

仍需补齐：
- Project
- AI Model
- Repository
- 文件名
- URL
- 技术术语（长尾）

说明：当前评分已可用，但实体类型覆盖尚未达到设计文档目标集合。

### 2.2 领域泛化能力仍偏弱（P1）

- 现状：以轻量召回为主，标签语义被弱化为附加信息。
- 问题：跨领域、新术语、复合短语的稳定性仍不足。
- 待办：增加“文本级实体归一规则层”（非词典补洞式）。

### 2.3 评估基准缺失（P1）

- 未建立固定测试集与回归指标。
- 待补：
	- 100~300 条标注样本（技术/非技术/噪声混合）
	- 指标：实体召回率、误报率、评分排序一致性
	- 回归脚本：每次改动自动输出对比报告

## 三、三阶段路由未落地项

### 3.1 Rule→SLM→LLM 的流量占比控制未实现（P0）

- 目标占比（规划）：Rule ~90%，SLM ~9%，LLM ~1%。
- 现状：尚无基于阈值的稳定分流闭环与监控面板。
- 待办：
	- 明确并固化阈值策略
	- 记录每阶段命中率与降本效果

### 3.2 Feature Score 维度未接入（P0）

- 未实现项：
	- 时间衰减
	- embedding 新颖性
	- 交互深度

### 3.3 Final Score 融合未接入（P0）

- 规划融合公式尚未落地。
- 待办：引入 rule/feature/semantic 加权融合与可配置权重。

---

## 四、SLM 成本与部署未完成

### 4.1 本地 SLM 替换未完成（P0）

- 现状：仍依赖远端模型调用。
- 目标：本地小模型承接中间层判断，降低在线成本与外部依赖。

### 4.2 Batch Aggregation 未实现（P1）

- 现状：逐条处理。
- 待办：N 条或 T 秒合并后再判定，降低调用次数并提升上下文一致性。

---

## 五、存储与检索未完成项

### 5.1 记忆字段结构增强（P1）

- tags 多值结构未落地。
- entity / compressed_from 等预留字段未完善。

### 5.2 检索排序融合未落地（P1）

- 现状偏单一相似度排序。
- 待办：importance + similarity + recency 融合排序。

### 5.3 缓存机制未实现（P2）

- memory_id -> semantic_score 缓存与失效策略未落地。
